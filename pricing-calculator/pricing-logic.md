## 1. Spend-to-Platform Fee (North Star)

The platform fee is determined by **spend under management** — the total annual spend entering the Aerchain platform. This is NOT total company spend; it is only the spend routed through Aerchain.

### Spend Tiers

| Spend (USD) | Platform Fee (USD) | BPS | Marginal BPS |
|-------------|-------------------|-----|-------------|
| $100M | $100,000 | 10.0 | — |
| $250M | $225,000 | 9.0 | 8.3 |
| $500M | $400,000 | 8.0 | 7.0 |
| $750M | $575,000 | 7.7 | 7.0 |
| $1,000M | $750,000 | 7.5 | 7.0 |

### Interpolation Rule

For spend values between tiers, **linear interpolation** is used between the two nearest tier boundaries.

- Below $100M: Extrapolate downward at 10 BPS (e.g., $50M → $50,000)
- Above $1B: Extrapolate at 7.0 BPS marginal rate (e.g., $1.5B → $750,000 + $500M × 7 BPS = $1,100,000)

## 2. Product Multipliers

Applied to the base platform fee. Only one base product; others are additive.

| Product | Multiplier | Description |
|---------|-----------|-------------|
| Intake to Award | 1.00x | Base — Autonomous & Strategic Sourcing |
| Procure to Pay | +0.75x | PO, Invoice, Payments |
| Spend Insights | +0.40x | Analytics & Intelligence |

**Combined multiplier** = sum of all selected product multipliers.

- Intake to Award only: 1.00x
- Intake to Award + P2P: 1.75x
- Intake to Award + Spend Insights: 1.40x
- Full Suite: 2.15x

**Formula**: `Platform Fee = Base Fee × Combined Product Multiplier`

---

## 3. Entity / Business Unit Multiplier

Accounts for multi-entity complexity (separate DOA matrices, supplier lists, policies).

| Entities / BUs | Multiplier |
|---------------|-----------|
| 1 | 1.00x |
| 2–3 | 1.15x |
| 4–10 | 1.30x |
| 10+ | 1.50x |

**Formula**: `Platform Fee = Base Fee × Product Multiplier × Entity Multiplier`

## 6. Subscription Composition

The total annual platform subscription is composed of:

```
Platform Subscription = Platform Base Fee + AI Workflow Fees + Integration Fees

Where:
  Platform Base Fee = interpolate(spend, spendTiers) × productMultiplier × entityMultiplier
  AI Workflow Fees  = Σ (channel_volume × per_txn_cost) across all active channels
  Integration Fees  = Σ (selected integration costs)
```

### Subscription Composition Display

Show as a stacked bar:
- **Platform**: Base fee portion
- **AI Workflows**: Transaction-based portion
- **Integrations**: Integration fee portion

## 11. TCO Calculation

### Year 1

```
Y1 Platform Subscription = Platform Fee (after all multipliers and discount)
Y1 Implementation = Implementation Total (rate card × tier multiplier) + Add-Ons
Y1 Total = Y1 Platform Subscription + Y1 Implementation
```

### Multi-Year

```
Yn Subscription = Y(n-1) Subscription × (1 + Escalation%)
Implementation is one-time (Year 1 only)
```

### 3-Year TCO

```
3-Year TCO = Y1 Total + Y2 Subscription + Y3 Subscription
```

### 5-Year TCO

```
5-Year TCO = 3-Year TCO + Y4 Subscription + Y5 Subscription
```

## 12. Auto-Derivation Rules

When the user provides minimal input, the system derives missing values using these rules. All derived values are flagged as **assumptions** in the proposal output.

### Minimum Required Input

1. **Customer Name** — identity
2. **Annual Spend (USD)** — drives all pricing
3. **Product Selection** — at least one product selected

### Derivation Chain

```
spend → client tier → default users, entities
spend → estimated txn volume → default channel split
spend → platform base fee (interpolated from tiers)
tier → implementation rate card (with tier multiplier)
defaults → integrations (1 standard ERP + SSO)
defaults → deal parameters (0% discount, 3yr, 10% escalation)
```

### Assumption Flags

Every derived value must be tagged in proposal output:

- `[ASSUMED]` — Value was auto-derived, not confirmed by user
- `[CONFIRMED]` — Value was explicitly entered or overridden by user
- `[CALCULATED]` — Value was computed from other inputs (confirmed or assumed)

## 14. Pricing Validation Rules

### Sanity Checks

1. Platform fee should be between 5–15 BPS of spend under management
2. Implementation fee should not exceed 100% of Y1 subscription
3. Total Y1 investment should be between 15–25 BPS of spend for full suite
4. Per-transaction costs should never go below the floor prices in the volume tier table
5. Deal discount cannot exceed 30% without executive override
